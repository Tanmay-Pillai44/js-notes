Call(), apply() and bind()

call() and apply():

Both sets this keyword inside the function and immediately executes that function. Only the difference is apply() methods accpets an array of arguments instead of comma seperated values.

bind():

It creates a new function and sets the this keyword to the specified object.

        function greeting () {
            console.log(`Hi, I am ${this.name} and I am ${this.age} years old`);
        }
        const john = {
            name: 'John',
            age 24
        }
        const jane = {
            name: 'Jane',
            age 22
        }

        function.call(thisArg, Arg1, Arg2);
        function.apply(thisArg, [agrumentsArr])

        Ex:
        greeting.call(john); // "Hi, I am John and I am 24 years old"
        greeting.apply(jane); // "Hi, I am Jane and I am 22 years old"

        const greetingJohn = greeting.bind(john);
        greetingJohn(); // "Hi, I am John and I am 24 years old"
        OR
        greeting.bind(john)();

Prototypes

Prototypes are objects from which other objects can inherit properties.

        // rabbit inherits from animal

        let animal = {
            eats: true,
            moves: true
        }

        let rabbit = {
            jumps: true,
            __proto__: animal
        }

Protoype Chaining

  1. Prototypes are the means of inheritance in JS.
  2. The Prototype of ana Object could also have prototype object and this continues until we reach the top level, where no prototype object is there.

This is called Prototype Chaining.

        // pocket -> bed -> table -> head

        let head = {
            glasses: 1
        }

        let table = {
            pen: 3,
            __proto__: head
        }

        let bed = {
            sheet: 1,
            pillow: 2,
            __proto__: table
        }

        let pockets = {
            money: 2000,
            __proto__: bed
        }
